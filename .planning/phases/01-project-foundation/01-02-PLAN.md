  <action>
Initialize Expo app in the mobile/ directory using create-expo-app with TypeScript template:

Run: `npx create-expo-app@latest mobile --template blank-typescript`

This creates:
- package.json with Expo dependencies
- tsconfig.json with TypeScript configuration
- App.tsx with basic component
- app.json with Expo configuration

After initialization, update mobile/app.json:
- Set "name" to "Noms"
- Set "slug" to "noms"
- Update "splash.backgroundColor" to "#FFF8E7" (warm cream from UI_SPEC.md)
- Update "backgroundColor" to "#F5F5F5" (light gray from UI_SPEC.md)

Create mobile/.env.example with:
```
# Backend API
EXPO_PUBLIC_API_URL=http://localhost:8000

# Google Maps (will be configured in Phase 7)
EXPO_PUBLIC_GOOGLE_MAPS_KEY=your_key_here
```

Update mobile/App.tsx to be a minimal "Hello Noms" screen with:
- SafeAreaView wrapper
- Centered text showing "Noms"
- Background color matching UI_SPEC (#F5F5F5)
- StyleSheet using exact spacing and typography from UI_SPEC (font-black, text-3xl equivalent)

DO NOT install additional packages yet (navigation comes in Phase 6). DO NOT create .env file (use .env.example only).

<task type="auto">
  <name>Task 2: Configure code quality tooling and documentation</name>
  <files>mobile/.prettierrc, mobile/.eslintrc.js, mobile/README.md</files>
  <action>
Create mobile/.prettierrc with opinionated config:
```json
{
  "semi": true,
  "trailingComma": "es5",
  "singleQuote": true,
  "printWidth": 80,
  "tabWidth": 2
}
```

Create mobile/.eslintrc.js extending Expo's config with TypeScript rules:
```js
module.exports = {
  extends: ['expo', 'prettier'],
  plugins: ['prettier'],
  rules: {
    'prettier/prettier': 'warn',
  },
};
```

Install prettier plugin: Add to mobile/package.json devDependencies:
```
"eslint-config-prettier": "^9.1.0",
"eslint-plugin-prettier": "^5.1.0",
"prettier": "^3.1.0"
```
Run: `cd mobile && npm install` after adding dependencies.

Create mobile/README.md with:
- Description: "Noms mobile app - React Native with Expo"
- Setup: npm install, copy .env.example to .env
- Development: npm start (or npx expo start)
- Platform commands: press 'i' for iOS, 'a' for Android
- TypeScript: npm run tsc for type checking
- Linting: Instructions for future (npx eslint .)
- Project structure overview (app/ for screens, components/ for reusable UI, coming in later phases)

Add npm scripts to mobile/package.json:
```
"scripts": {
  "start": "expo start",
  "android": "expo start --android",
  "ios": "expo start --ios",
  "web": "expo start --web",
  "tsc": "tsc --noEmit"
}
```
